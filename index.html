§ 01 — System Architecture

Six layers, narrow interfaces.

Each layer is independently testable but composes into one end-to-end mission — from cold boot at the field edge, through localization, planning, and inspection, to a complete pass over a target orchard block.

L0

Drive Controller

Twin-loop PID — linear velocity + heading

Tracks kinematic setpoints by closing the loop on left/right wheel velocities. The inner loop of the entire stack.

10 randomized slip/friction runs benchmark suite green
PID closed-loop
L1

Worker Tracker

Constant-velocity Kalman filter

Tracks moving agents detected as lidar clusters; predicts each track 1 second ahead so the planner can inflate cost.

tgt < 0.20 m, ≥ 95% assoc → 0.037 m, 100% assoc
5.4× vs target
L2

Global Localizer

Monte-Carlo localization — KLD adaptive resampling

Recovers rover pose at startup or after GNSS dropout using lidar scan matching against a coarse prior orchard map.

tgt < 0.15 m p95, ≤ 3000 particles → 0.029 m p95, ≤ 169 particles
5.2× vs target
L3

Pose-Graph SLAM

EKF-SLAM with tree-trunk landmarks · RANSAC + Mahalanobis gating

Continuously fuses odometry and lidar-detected trunks to refine rover pose and the orchard map online.

tgt < 0.20 m pose, < 0.30 m landmarks → 0.032 m pose, 0.070 m landmarks
6.3× vs target
L4

Kinematic Planner

Hybrid A* on (x, y, θ) lattice + analytic bounded-curvature connector

Plans a kinematically feasible path on the SLAM cost map, inflated by Kalman-predicted worker positions from L1.

tgt ≤ 250 ms, ≥ 30% steering reduction → 28.5 ms, 86.6% reduction
8.8× vs target
L5

Inspection Scheduler

Markov decision process — value iteration

Chooses which orchard row to inspect next under battery state-of-charge, daylight, and row-priority constraints.

tgt ≤ 5% oracle gap, < 800 ms → 0.000% gap, well under budget
∞ oracle-equiv.

§ 05 — Recruiter-Ready Summary

Drop straight onto the resume.

TerraScout — Built a six-layer autonomy stack (PID → Kalman → MCL → Pose-Graph EKF-SLAM → Hybrid A* → MDP) for a simulated GPS-degraded crop-inspection rover; completed 10/10 priority rows with 0.201 m mean pose error in 12 s wall-clock across 20 randomized 30-row missions.

  • Architected and implemented a six-layer Python autonomy stack for a simulated differential-drive crop-inspection rover, integrating PID drive control, Kalman-filter worker tracking, particle-filter localization, pose-graph EKF-SLAM, Hybrid A* kinematic planning, and MDP inspection scheduling — all behind clean module APIs.
  • Designed a Kalman-filter tracker for up to 10 simultaneous moving agents from lidar clusters; achieved 0.037 m mean 1-second-ahead prediction error and 100% data-association accuracy across 100 simulated scenes.
  • Implemented an adaptive Monte-Carlo localizer with KLD resampling that converges from a ±5 m / ±30° prior to 0.029 m p95 pose error using ≤ 169 particles — a 5.2× margin over the 3,000-particle, 0.15 m target.
  • Built a pose-graph EKF-SLAM module with Mahalanobis-gated data association on lidar-detected tree trunks; achieved 0.032 m drift-corrected pose and 0.070 m landmark error after 5-minute traversals across procedurally generated 12×30-row orchards.
  • Developed a Hybrid A* planner with bounded-curvature analytic connectors on a SLAM cost map inflated by Kalman-predicted worker positions; cut total steering effort by 86.6% vs. baseline grid A* with mean solve time 28.5 ms.
  • Solved orchard inspection scheduling as an MDP via value iteration over battery-SOC and time-of-day state; computed policy reaches the oracle exactly (0.00% gap) across 50 randomized layouts under battery + daylight constraints.
  • Engineered a reproducible benchmark + CI pipeline (ruff, mypy --strict, pytest, 90%+ coverage) that auto-publishes KPI tables to the project README on every push to main.
§ 06 — About

Built by Kazmir Fahrier.

A self-directed twelve-week capstone designed to demonstrate full-stack autonomy competence for robotics and ML engineering roles. The project pulls from foundational textbook techniques — Probabilistic Robotics (Thrun, Burgard & Fox), Planning Algorithms (LaValle), Feedback Systems (Åström & Murray), Sutton & Barto — and integrates them into one runnable, reproducible, end-to-end stack.

Every line is clean-room original code. No course-provided starter, simulator, test fixtures, or assets. The vehicle class (skid-steer ground rover) and domain (precision agriculture / GPS-degraded orchard inspection) were chosen specifically to keep the work independent of any prior coursework.

Author background: BSc Electrical & Electronic Engineering (BUET, US-equivalency credentialed), Georgia Tech OMSCS in progress, hands-on AI evaluation work as an AI trainer with deep coverage of analog/digital circuits, control systems, and communications.
